Getting Started

Zerv is a web framework that offers no-hassle templating and app creation.

First install using Yarn or npm:

Yarn:

yarn add @zerv/core

NPM:

npm install -g parcel-bundler

Then create your app webpage. Wherever you want a value to be replaced by your data, reference the path to it within {‍{ and }}

<html>
<!-- head ... -->
<body>
<div id="app">
    <h1>Welcome to {‍{name}}</h1>
</div>
<script src="app.js"></script>
</body>
</html>

And create your app.js file:

import Zerv from "@zerv/framework";
Zerv.disableDebugLogging();

const app = new Zerv("#app");
app.data.name = "My First App";

app.render();

Build your app using a bundler like Parcel:

parcel build index.html

Template Engine Basics

The template engine can do lots more than simply replacing values in a file.

How to use the template engine

If you want to use it in the text content of an element, surround the name of the variable you want to link to with double squigly brackets (i.e. {‍{ and }}). However, this will not work for attributes. Instead, prepend the attribute with z-bind:, and it will act the same as templates in text source.

Live Value Updates

Add a template value to your webpage:

<p>Welcome, {‍{user.name}}</p>

What happens if the user wants to change their username? Instead of reloading the page or putting the name in an element, you can simply update the data value:

// before render:
app.data.user = {
    name: "Jeff007"
};

// when they change their username:
app.data.user.name = "Xx_JEFF_xX";

The value will automatically update!

Javascript Expressions

Javascript expressions can be used inside template values, just as they would in normal Javascript. This allows for you to modify the value without having to create more values in the data.

<p>Multiplication result: {‍{doubler.input * 2}}</p>

If you're scared that this internally uses eval, don't be - its scope is limited to the data you input, and it doesn't support anything dangerous, as it is using a custom evaluator. To find out what it does support, go here.

Input linkage

Most types of inputs can be linked to a data value using the z-bind attribute (note, this is different to prepending an attribute with z-bind:). Set the value of the attribute to the name of the variable you want to link it to - now, any changes in the input will update the variable, however currently changes in the variable will not update the input.

The z-bind attribute supports most types of inputs, as well as content-editable elements and textareas.
